UML Tool Report: PlantUML

Tool

PlantUML — текстовий інструмент для створення UML-діаграм, який генерує діаграми з коду (DSL) у PNG/SVG.

Why I chose it

  • Швидко створювати багато діаграм і підтримувати їх актуальними.
  • Діаграми легко версіонуються в Git (звичайні текстові файли).
  • Зручно робити однаковий стиль і повторювані структури (пакети, стани, sequence).
  • Експорт у PNG/SVG підходить для звіту та перегляду без спеціальних програм.

Supported diagrams I used

  • Use Case
  • Class diagram (domain and code-level)
  • Sequence diagrams
  • Activity diagrams
  • State machine
  • Component/package view

Workflow

  1. Створюю .puml у репозиторії.
  2. Перевіряю прев’ю (web editor або VS Code extension).
  3. Експортую PNG/SVG і зберігаю у папку images/.

Pros

  • “Діаграми як код”: зручно підтримувати й оновлювати.
  • Добре працює з Git, легко бачити diff.
  • Швидкий експорт у PNG/SVG.
  • Підходить як для AS-IS, так і для TO-BE (до/після рефакторингу).

Cons

  • Потрібно звикнути до синтаксису.
  • Для дуже великих діаграм інколи потрібно вручну керувати розташуванням елементів.
  • Для деяких нотацій UML може знадобитися пошук правильного синтаксису або додаткові налаштування.

Conclusion

PlantUML підходить для навчальних UML-завдань і для реальних проєктів, де важлива підтримка діаграм у актуальному стані та зручне версіонування.
